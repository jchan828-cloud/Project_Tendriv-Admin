What kind of dynamic resizing?

The current shell has a fixed 200px sidebar and 44px topbar. Pick the resizing behaviour that fits your workflow.

A

Collapsible sidebar

A toggle button collapses the sidebar to a narrow icon rail (or fully hidden), giving the main content more room. One click to expand, one to collapse. Works at any screen size.

Draggable sidebar splitter

A drag handle between the sidebar and main content lets the user freely resize the sidebar (e.g. 160px–400px). Width is persisted to localStorage so it survives page reloads.

Responsive breakpoints only

No manual resizing — the layout adapts automatically to the browser window width. On wide screens the sidebar is full width; on narrower screens it collapses into a slide-over drawer (hamburger menu).
