Design — Section 1: Architecture

How the shell layout changes with react-resizable-panels

Current shell (CSS Grid, fixed widths)
// app/(dashboard)/layout.tsx — server component
<div className="shell"> ← CSS grid: 200px 1fr / 44px 1fr
  <Topbar />
  <Sidebar />
  <main>{children}</main>
</div>
New shell (flex column + PanelGroup, dynamic widths)
// app/(dashboard)/layout.tsx — server component (unchanged data fetching)
<div className="shell"> ← flex column, 100vh
  <Topbar /> ← row 1, 44px fixed
  <ResizableShell sidebar=<Sidebar .../>> ← NEW client component
    {children} ← passed as children slot (server RSC boundary preserved)
  </ResizableShell>
</div>
ResizableShell internals
// components/layout/resizable-shell.tsx — 'use client'
<PanelGroup direction="horizontal" autoSaveId="tendriv-shell">
  <Panel defaultSize={15} minSize={10} maxSize={25} collapsible>
    {sidebar} ← slot prop — server-rendered Sidebar
  </Panel>
  <PanelResizeHandle /> ← styled jade drag handle
  <Panel>
    <main>{children}</main> ← toggle button lives here too
  </Panel>
</PanelGroup>
Key RSC boundary insight
The server component boundary is preserved. DashboardLayout stays a server component — it fetches the user role and passes <Sidebar /> as a prop to the client ResizableShell. Server-rendered children flow through as a slot. No data fetching moves to the client.
